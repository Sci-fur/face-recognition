## Face Recognition System

### Version 1.0

## 1. Introduction

### 1.1 Purpose
This Software Requirements Specification document describes the functional and non-functional requirements for a **Real-Time Face Recognition System**. The system uses a laptop webcam (the peripheral device) to detect and recognize human faces in real-time by comparing them against a pre-trained dataset of known individuals. The primary goal is to achieve precise and reliable person identification.

### 1.2 Scope
The Face Recognition System is a terminal-based application that:
- Captures face images via a webcam to build a training dataset
- Uses deep learning (PyTorch + facenet-pytorch) to extract facial feature embeddings
- Trains a machine learning classifier (SVM) on the extracted features
- Performs real-time face recognition from a live webcam feed
- Displays recognized names and confidence scores on the video output

The system is scoped to 5–10 known individuals for this academic project.

### 1.3 Definitions, Acronyms, and Abbreviations

| Term | Definition |
|------|------------|
| **Face Detection** | Locating human faces in an image/video frame |
| **Face Alignment** | Normalizing a detected face to a standard pose and scale |
| **Face Embedding** | Numerical vector representation of a face generated by a neural network |
| **MTCNN** | Multi-Task Cascaded Convolutional Neural Network — a face detection model |
| **InceptionResnetV1** | A deep convolutional neural architecture for face recognition, pretrained on VGGFace2 |
| **SVM** | Support Vector Machine — a supervised learning classifier |
| **VGGFace2** | A large-scale face recognition dataset used for pretraining |
| **FPS** | Frames Per Second — measure of real-time performance |
| **CLI** | Command-Line Interface |

## 2. Overall Description

### 2.1 Product Perspective
The system is a self-contained Python application designed for a university **Computer Peripheral Lab** project. The webcam serves as the primary peripheral input device. The system processes the webcam's video stream to perform facial recognition and outputs labeled video frames to the user's display.

### 2.2 Product Functions
The system provides three primary functions accessible via CLI commands:

1. **Data Collection (`collect`)** — Captures face images of a person via webcam, with automatic face detection, alignment, and storage in a structured dataset directory.

2. **Training (`train`)** — Reads the collected dataset, detects and aligns faces, extracts deep learning embeddings using InceptionResnetV1, trains an SVM classifier, and saves the trained model to disk.

3. **Recognition (`recognize`)** — Opens a live webcam feed, detects faces in each frame, extracts embeddings, classifies them against the trained model, and overlays the person's name and confidence on the video stream.

### 2.3 User Characteristics
- **End Users**: University students and instructors
- **Technical Level**: Basic familiarity with terminal commands
- **Use Case**: Demonstrate real-time face recognition using a webcam peripheral

### 2.4 Operating Environment
- **Operating System**: Linux (Ubuntu 20.04+), Windows 10+, or macOS 11+
- **Runtime**: Python 3.10 or higher
- **Hardware**: Laptop with built-in webcam (minimum 720p) or external USB webcam

### 2.5 Design and Implementation Constraints
- **C-01**: Must use the webcam as the primary peripheral input device
- **C-02**: Must use deep learning (PyTorch/facenet-pytorch) for face feature extraction
- **C-03**: Must operate as a CLI application (no GUI framework)
- **C-04**: Must work with a dataset of 5–10 individuals
- **C-05**: Must achieve real-time performance (≥10 FPS) on standard laptop hardware

### 2.6 Assumptions and Dependencies
- **A-01**: The webcam is functional and properly connected/driven
- **A-02**: Sufficient lighting for face detection (normal indoor lighting)
- **A-03**: The person's face is mostly visible (not heavily occluded)
- **A-04**: Internet connection is available for downloading pretrained model weights on first run
- **A-05**: PyTorch with CUDA is optional; the system runs on CPU if no GPU is available

## 4. System Architecture and Data Flow

### 4.1 Architecture Overview
The system follows a modular pipeline architecture with four main components:

```
┌─────────────────────────────────────────────────────────┐
│                     app.py (CLI Entry Point)              │
├─────────────┬─────────────────┬──────────────────────────┤
│  collect.py  │    train.py     │      recognize.py        │
│  (FR-01)     │   (FR-05)       │      (FR-07, FR-08)      │
├──────┴──────┴─────────────────┴──────────────────────────┤
│                    Core Modules                           │
├─────────────┬─────────────────┬──────────────────────────┤
│ detector.py │   embedder.py    │    classifier.py         │
│  (FR-02,03)  │   (FR-04)       │    (FR-05.3, FR-06)      │
└─────────────┴─────────────────┴──────────────────────────┘
```

### 4.2 Data Flow

#### Data Collection Flow
```
Webcam → Frame → MTCNN Detection → Face Alignment → Save to dataset/<name>/
```

#### Training Flow
```
dataset/*.jpg → MTCNN Detection+Alignment → InceptionResnetV1 → 512-d Embedding
                                                                      ↓
                                                            SVM Training (80/20 split)
                                                                      ↓
                                                          model/*.pkl (SVM+Scaler+Classes)
```

#### Recognition Flow
```
Webcam → Frame → [Every Nth frame] → MTCNN Detection+Alignment → InceptionResnetV1
                                                                         ↓
                                                                  512-d Embedding
                                                                         ↓
                                                          SVM Classifier (loaded from model/)
                                                                         ↓
                                                      Bounding Box + Name + Confidence
                                                                         ↓
                                                              Display Output Frame
```

## 5. Test Cases and Validation

### 5.1 Unit Test Cases

| TC-ID | FR-ID | Test Case | Expected Result |
|-------|-------|-----------|-----------------|
| TC-01 | FR-01 | Run `collect` with `--name "Alice"` | Webcam opens, images saved to `dataset/Alice/` |
| TC-02 | FR-01 | Run `collect` with no name | Error message: name is required |
| TC-03 | FR-02 | Image with one face | Face detected, bounding box returned |
| TC-04 | FR-02 | Image with no face | No face detected, empty result |
| TC-05 | FR-04 | Feed aligned face to InceptionResnetV1 | Returns 512-d tensor |
| TC-06 | FR-05 | Train with 3+ persons in dataset | SVM trained, model files saved |
| TC-07 | FR-05 | Train with empty dataset | Error message: no data found |
| TC-08 | FR-06 | Load saved model files | SVM, scaler, classes loaded without error |
| TC-09 | FR-07 | Run `recognize` with trained model | Webcam opens, faces labeled in real-time |
| TC-10 | FR-09 | Show unknown face to camera | Displayed as "Unknown" |

### 5.2 System Validation Metrics
- **Accuracy**: (True Positives + True Negatives) / Total Predictions ≥ 90%
- **Precision**: True Positives / (True Positives + False Positives) ≥ 90%
- **Recall**: True Positives / (True Positives + False Negatives) ≥ 85%
- **F1-Score**: 2 × (Precision × Recall) / (Precision + Recall) ≥ 87%

### 5.3 Testing Procedure
1. Collect face data for 5 persons (50 images each = 250 images total)
2. Train the model
3. Have each person sit in front of the webcam for 30 seconds
4. Record recognition results (correct, incorrect, unknown)
5. Calculate accuracy, precision, recall, and F1-score
6. Test with 2 unknown persons to check rejection capability

### Appendix C: Known Limitations
1. Recognition accuracy decreases in poor lighting conditions
2. Extreme head poses (profile view) may fail detection
3. The system does not perform re-identification across sessions (no database)
4. Performance degrades with >3 faces in a single frame
5. First run requires downloading model weights (~200 MB)
